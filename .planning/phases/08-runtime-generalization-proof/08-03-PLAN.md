# P8-03 Plan - Add Adapter-Backed Landing-Gear Diagnosis Proof

## Objective

Prove the new second-system adapter can traverse the existing fault-diagnosis contract, so runtime generalization reaches beyond nominal playback into reusable fault reasoning.

## Context

- `P8-02` already proved the landing-gear adapter can feed the playback contract and stay aligned with sampled adapter truth.
- The existing diagnosis layer is still wired through intake/spec entry points, so the next safe slice is an adapter-backed helper plus a landing-gear fault proof rather than a broader architectural rewrite.
- Landing-gear fault modes already exist in the adapter-published spec (`hydraulic_pressure_bias_low`, `uplock_stuck_locked`), which makes this a direct runtime-proof extension of the current work.

## Tasks

1. Add the smallest adapter-backed diagnosis entry point that can reuse the published fault-diagnosis contract.
2. Validate a landing-gear fault scenario end to end and compare the resulting blocked logic / completion outcome against expected runtime truth.
3. Add focused/shared validation for the adapter-backed diagnosis proof before Notion sync.

## Success Criteria

- A non-reference adapter can produce a schema-valid fault-diagnosis report through the existing diagnosis contract.
- The landing-gear diagnosis proof shows a clear baseline-vs-fault divergence and expected blocked logic under the chosen fault mode.
- The default shared suite now guards this adapter-backed diagnosis proof against regression.
