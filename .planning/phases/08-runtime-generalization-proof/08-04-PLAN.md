# P8-04 Plan - Add Adapter-Backed Landing-Gear Knowledge Proof

## Objective

Extend the runtime-generalization proof through the knowledge layer, so a second-system adapter can now produce reusable incident/repair artifacts through the same contract chain.

## Context

- `P8-01` through `P8-03` proved a landing-gear adapter can publish truth metadata/spec, drive playback, and produce schema-valid diagnosis artifacts.
- The remaining gap in the main adapter runtime chain is knowledge capture, which is still wired through intake-only entry points.
- This is a small safe slice because the existing knowledge contract already depends on diagnosis output; it just needs an adapter-backed helper plus a focused proof.

## Tasks

1. Add the smallest adapter-backed knowledge-artifact entry point that reuses the published diagnosis and knowledge contracts.
2. Validate a landing-gear resolved-incident artifact end to end and prove the resulting payload stays schema-valid.
3. Add focused/shared validation for the adapter-backed knowledge proof before Notion sync.

## Success Criteria

- A non-reference adapter can produce a schema-valid knowledge artifact through the existing knowledge contract.
- The landing-gear proof yields a resolved knowledge artifact with the expected diagnosis chain embedded.
- The default shared suite now guards this adapter-backed knowledge proof against regression.
