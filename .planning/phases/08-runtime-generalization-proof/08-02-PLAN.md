# P8-02 Plan - Add Adapter-Backed Landing-Gear Playback Proof

## Objective

Prove the new second-system adapter can drive the existing playback contract end to end, so runtime generalization is no longer limited to metadata/spec publication.

## Context

- `P8-01` introduced a minimal landing-gear controller adapter plus a generic truth-adapter layer and added that proof to the shared validation suite.
- The current playback compiler is already spec-driven, but its runtime proof still comes from intake/spec fixtures rather than a real second-system adapter.
- Landing-gear playback also exposes a generalization edge: categorical states such as `UP/DOWN` must compile into playback logic without forcing the contract back into a reference-system-only numeric shape.

## Tasks

1. Add the smallest adapter-backed playback entry point that can compile a trace from an adapter-published control-system spec.
2. Make the playback compiler safely handle the landing-gear discrete-state shape and compare sampled playback checkpoints against adapter truth.
3. Add focused/shared validation for the landing-gear playback proof before Notion sync.

## Success Criteria

- A non-reference adapter can produce a playback trace through the published playback contract.
- The landing-gear playback trace stays schema-valid and aligns with adapter truth at representative checkpoints.
- The default shared suite now guards this adapter-backed playback proof against regression.
