# P8-01 Plan - Implement Minimal Landing-Gear Controller Adapter

## Objective

Add the first real non-thrust-reverser runtime truth adapter, so P8 can prove the new generalized contract layer is capable of hosting a second system without touching `controller.py`.

## Context

- P7 closed with the contract/schema layer complete through `P7-70`, including root spec schema, fault taxonomy, controller-truth metadata, playback trace, bundle, archive manifest, and second-system smoke schema validation.
- The current `ControllerTruthAdapter` protocol is still shaped around the reference thrust-reverser runtime, so P8 needs a backward-compatible generic truth-adapter layer rather than a breaking rewrite of the demo or runner.
- The user explicitly approved the constitution boundary update: new system truth is allowed only through adapter interfaces, and bypassing adapters with new hardcoded truth paths is forbidden.

## Tasks

1. Extend the adapter layer with a backward-compatible generic runtime-truth interface that can publish a spec payload and evaluate a generic snapshot.
2. Implement a minimal landing-gear controller adapter that exposes valid metadata/spec and deterministic nominal-vs-blocked truth evaluation.
3. Add focused regression and validation coverage, then re-run focused/shared validation before Notion sync.

## Success Criteria

- The repo contains a non-reference controller adapter that does not depend on `DeployController`.
- The landing-gear adapter publishes schema-valid metadata and a schema-valid control-system spec through the adapter boundary.
- Deterministic snapshot evaluation proves the second system can emit runtime truth without touching the reference thrust-reverser path.
