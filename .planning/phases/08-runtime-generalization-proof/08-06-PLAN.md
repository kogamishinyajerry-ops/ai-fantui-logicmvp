# P8-06 Plan - Add A Two-System Adapter-Backed Runtime Comparison Report

## Objective

Prove that runtime generalization is no longer a single-adapter story by emitting one machine-readable comparison report that summarizes the adapter-backed proof chain for both the reference thrust-reverser system and the landing-gear system.

## Context

- `P8-05` moved the default second-system smoke entrypoint onto the landing-gear adapter-backed runtime chain.
- The next smallest runtime-generalization slice is not UI polish or adapter registry work; it is to compare the reference adapter and the second-system adapter through one shared proof artifact.
- This should stay contract-focused and reuse the already-published playback, diagnosis, and knowledge builders instead of inventing a new runtime path.

## Tasks

1. Add a reusable two-system runtime comparison report that summarizes the adapter-backed proof chain for the reference thrust-reverser adapter and the landing-gear adapter.
2. Publish schema validation and focused tests for that comparison report, including a CLI entrypoint.
3. Promote the comparison validator into the shared suite so the default regression baseline guards the new two-system proof.

## Success Criteria

- A single machine-readable report can compare the reference and landing-gear adapter-backed runtime proof chains.
- The comparison payload is schema-valid and proves both systems complete playback, block the selected fault path, and emit resolved knowledge artifacts.
- The shared validation suite includes the new two-system comparison proof.
