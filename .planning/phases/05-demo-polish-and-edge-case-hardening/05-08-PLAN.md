## Objective

Repair the browser-side live-control wiring so VDT adjustments once again produce fresh lever snapshots, update on-screen parameter readouts immediately, and reopen the conditional TRA deep-reverse band whenever the `L4` boundary logic is satisfied.

- `P5-07` clarified the intended drag-band semantics, but moving the VDT controls into the top lever deck introduced a frontend regression.
- The moved `feedback_mode` and `deploy_position_percent` controls were no longer included in the snapshot-rescheduling selector, so dragging VDT stopped sending fresh `POST /api/lever-snapshot` requests.

- This is a real blocker for the live cockpit and must be fixed before the current P5 review target is trustworthy.

## Tasks

1. Reconnect the moved live controls to the same browser-side snapshot scheduling path used by the rest of the cockpit inputs.
2. Keep the current conditional TRA drag semantics intact while restoring immediate VDT parameter updates and deep-range unlock behavior.
3. Extend regression coverage so future layout moves cannot silently detach the live-control inputs from snapshot recomputation again.

## Success Criteria

- Dragging `VDT` updates the visible percentage readout and recomputes the current lever snapshot immediately.
- When the VDT override satisfies the `L4` boundary logic, the browser reopens the full `-32° .. 0°` TRA free-drag range again.
- The existing `POST /api/lever-snapshot` contract and backend gate semantics remain unchanged.
- Shared validation and demo smoke both pass with the repaired live-control wiring included in the automated evidence plane.
