# P5-10 Plan - Add Controlled State-Vs-Time Monitoring Timeline

## Objective

Add a deterministic `状态 vs 时间` monitoring view that charts the current logic panel as a full timeline, using the user-defined `RA -> TRA -> VDT` sequence while keeping the existing cockpit layout readable.

## Context

- The current cockpit already shows a single live snapshot, but it does not let the user inspect how the whole chain evolves over time.
- The user provided a specific monitoring script: `RA` falls from `7.0 ft`, `TRA` starts moving once `RA=6.0 ft`, `TRA` stops at `-14°`, and `VDT` then ramps from `0%` to `100%`.
- This timeline must not be a front-end-only fake chart. It should reuse the current switch + controller truth and expose a backend-generated evidence surface.
- The new monitor should not make the existing lever / condition / logic board area more crowded. It needs its own roomy panel.

## Tasks

1. Add a deterministic monitoring timeline generator and API that evaluates the current controller chain against the user-defined `RA -> TRA -> VDT` sequence.
2. Add a dedicated full-width monitor panel that renders a multi-row `状态 vs 时间` chart with key event markers and improved spacing.
3. Extend regression coverage for the new API contract, key transition times, and static monitor assets without breaking the existing demo surface.

## Success Criteria

- A backend endpoint returns deterministic time-series data for the monitored chain, including `RA`, `TRA`, switch states, logic gates, power/command outputs, `VDT`, and `THR_LOCK`.
- The UI renders the monitoring chart in a separate panel, preserving the readability of the existing cockpit controls and logic board.
- Key timeline checkpoints such as `t=10s`, `t=24s`, `t=42s`, `t=44s`, and the `RA=0 ft` hold segment are regression-protected.
- Unit/static regressions, demo smoke, and the shared validation suite all pass with the new monitoring layer in place.
