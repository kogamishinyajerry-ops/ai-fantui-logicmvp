# P5-01 Plan - GitHub-Verifiable Demo Smoke Suite

## Objective

Create a GitHub-verifiable smoke suite for the presenter demo path so P5 begins by replacing vague browser-confidence checks with automated evidence around edge-case interactions and payload switches.

## Context

- P4 is approved and the presenter-ready surface is stable.
- P5 now prioritizes edge-case hardening and smoke confidence, not new UI expansion.
- The current validation loop already proves unit and schema contracts, but it does not yet run a dedicated demo-path smoke suite through the HTTP layer.
- The smoke suite must stay within the current review contract: GitHub Actions + Notion evidence, no browser-only approval path, no second control-truth layer.

## Tasks

1. Add a lightweight demo smoke script that exercises the local HTTP demo surface across the highest-value edge cases:
   - controlled `POST /api/demo` prompt path
   - clamped extreme lever inputs on `POST /api/lever-snapshot`
   - payload/mode switches between auto scrubber and manual feedback override
   - one expected invalid-input error response
2. Add regression tests for the smoke script and wire it into the shared validation suite so GitHub Actions produces this evidence automatically.
3. Update local control-plane state so `P5-01` is the active default plan and the repo documents this new smoke evidence surface.

## Success Criteria

- A single command can run the demo smoke suite locally and in GitHub Actions.
- The smoke suite proves the demo HTTP surface handles extreme/clamped values, mode switching, and expected error paths without browser automation.
- The shared validation suite includes the smoke suite and still passes end-to-end.
- Local GSD state and Notion control-plane routing point at `P5-01`.
