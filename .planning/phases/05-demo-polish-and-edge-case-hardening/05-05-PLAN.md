# P5-05 Plan - L4-Gated TRA Lock And Compact Cockpit Layout

## Objective

Align the live demo cockpit with the requested reverse-travel lock behavior and reduce the left-column vertical sprawl so VDT adjustment and the right-side logic board stay readable together.

## Context

- `P5-04` finished smoke coverage for the visible blocker toggles, but the live cockpit still had two usability gaps.
- The TRA slider currently allows deep reverse requests immediately, even though the requested interaction model should lock deeper rear travel behind an `L4`-ready gate at `-14°`.
- The left-side lever and condition stack is tall enough that adjusting `VDT` can push the logic board out of view on desktop, which hurts presenter flow.
- The project constitution still forbids introducing a second control-truth layer, so the lock must be enforced through the existing demo evidence path rather than UI-only tricks.

## Tasks

1. Enforce a backend `-14°` rear-travel lock that only releases when an `L4`-ready probe is satisfied, and surface the lock state back to the browser payload.
2. Update the cockpit shell so the TRA lock state is visible and the left-side lever/condition area is denser while preserving the existing demo/API contract.
3. Extend regression coverage and demo smoke so the new lock gate and compact cockpit behavior are GitHub-verifiable.

## Success Criteria

- Deep reverse requests are capped at `-14°` until the `L4` gate is satisfied, even if the request is sent directly to `POST /api/lever-snapshot`.
- The browser shell visibly communicates whether the TRA range is still locked or already unlocked.
- The left-side cockpit layout keeps VDT adjustment and the right-side logic board in a more usable same-screen flow on desktop.
- Shared validation and demo smoke both pass with the new lock behavior included in the automated evidence plane.
