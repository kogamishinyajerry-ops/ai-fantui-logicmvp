# P6-10 Plan - Make Dashboard-Only Degraded Mode Explicit

## Objective

Turn the current partial Notion-health reality into an explicit, validated operating mode instead of letting the control plane pretend every active subpage is healthy.

## Context

- `P6-09` made repo-side docs honest about archived `status / 09C / freeze` pages, but the validation layer still reports a plain pass as long as the dashboard itself remains reachable.
- The live dashboard is currently the canonical Notion surface, while the separate active subpages are still observed as archived by the local integration.
- The control plane should keep running in that state, but it should name it clearly as a dashboard-only degraded mode and stop rendering dead subpage entrypoints from the dashboard snapshot itself.

## Tasks

1. Teach `validate_notion_control_plane.py` to distinguish between a hard failure and a dashboard-only degraded mode.
2. Update dashboard snapshot rendering so it explicitly announces degraded mode and omits dead status/09C/freeze links.
3. Re-run validation and refresh the dashboard so the visible Notion surface matches the real health model.

## Success Criteria

- The validator exits successfully when only `status / 09C / freeze` are archived, but reports degraded mode explicitly in both text and JSON output.
- The validator still fails if the dashboard or a required database becomes unavailable or archived.
- The dashboard snapshot no longer advertises dead status-style subpage links while degraded mode is active.
