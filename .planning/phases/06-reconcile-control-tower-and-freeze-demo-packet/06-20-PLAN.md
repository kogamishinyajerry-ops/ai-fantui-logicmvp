# P6-20 Plan - Keep Active Sync Page URLs Stable Across Snapshot Refreshes

## Objective

Stop the active status / 09C / freeze pages from rotating to new Notion page IDs on every snapshot refresh when the current page is still writable.

## Context

- `P6-18` fixed active-page link drift and `P6-19` removed the last stale bootstrap examples, but the control plane still churns page IDs because `replace_active_sync_page()` creates a fresh child page whenever the rendered content changes.
- That means even a normal plan advance can invalidate recently shared `01 当前状态` / `09C` / `10 Freeze Demo Packet` URLs, despite the existing pages still being writable.
- We only need page rotation when Notion has archived, trashed, or lost the current active page; otherwise an in-place rewrite is enough and keeps the control-plane entrypoints stable.

## Tasks

1. Update `replace_active_sync_page()` so it rewrites the current page in place when the page exists and is writable but the rendered content has changed.
2. Keep the existing replacement path only for missing/archived pages, so the dashboard can still heal broken active surfaces automatically.
3. Add regression coverage for the in-place update path and the archived-page replacement path.

## Success Criteria

- Normal active-page refreshes keep the same page ID when the existing page is still writable.
- Archived or missing pages still get replaced with a fresh active page.
- Tests cover both behaviors and the control plane can continue refreshing without URL churn on every run.
