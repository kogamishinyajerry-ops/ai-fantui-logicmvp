# P6-18 Plan - Refresh Active Sync Surfaces When Link Targets Move

## Objective

Make the repo-side Notion sync refresh active status/freeze/brief pages whenever a linked target page changes, even if the visible label text stays the same.

## Context

- The active control-plane pages now live on replacement page IDs created by repo-side sync when archived ancestors block writes.
- `replace_active_sync_page()` currently decides whether a page is already up to date by comparing only visible rich-text content.
- That means a status or dashboard page can keep linking to an old `freeze_packet` / `status` / `opus_brief` page after the config was updated, because the link label is still `10 Freeze Demo Packet` or `01 当前状态（自动同步）`.
- The repo also still contains an early Notion bootstrap doc with historical example baselines like `Round 92` and `129 tests OK`; those examples should stay as setup history, not read like current truth.

## Tasks

1. Teach active-page render comparison to include rich-text link targets so stale Notion page URLs trigger a refresh.
2. Update the fallback default active-page IDs to the current live `status`, `09C`, and `freeze_packet` targets.
3. Add regression coverage for link-target drift and clearly label the old bootstrap Notion guide as historical/reference-only.

## Success Criteria

- A changed Notion page link target causes `page_matches_rendered_blocks()` to return false even when the rendered label text is unchanged.
- The sync script's default active-page IDs match the current live control-plane pages.
- The historical bootstrap Notion guide no longer reads like current baseline truth.
