# P6-21 Plan - Surface Database-Degraded Evidence Mode Across The Control Plane

## Objective

Make the control plane explicitly state when live shared Notion databases are unavailable and the current truth has been recovered from active pages or repo docs instead.

## Context

- `P6-20` stabilized the URLs for the active status / 09C / freeze pages, so the main remaining operational ambiguity is no longer link churn.
- The local integration still cannot query several shared Notion databases and returns `HTTP 404 /v1/databases/...`, but the automation can already recover current truth from active pages and repo docs.
- That fallback works operationally, yet the rendered dashboard / status / freeze / 09C / repo-handoff surfaces do not clearly tell readers which evidence tier produced the current snapshot.

## Tasks

1. Extend `ReviewSnapshot` with explicit evidence-mode metadata that distinguishes shared-database live reads from active-page, dashboard-only, repo-doc, and local-timeout fallback paths.
2. Render that evidence-mode line into the dashboard, status page, freeze packet, 09C review brief facts, and repo-side coordination docs.
3. Add regression coverage for the new metadata and the new degraded-mode copy so future control-plane cleanups cannot silently remove it.

## Success Criteria

- Every fallback snapshot carries a machine-readable evidence mode plus a human-readable explanation.
- Dashboard / status / freeze / 09C / repo docs visibly explain when the current truth is coming from fallback evidence rather than live shared databases.
- Tests cover the new copy and all targeted control-plane regression suites still pass.
