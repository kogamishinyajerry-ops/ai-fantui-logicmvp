# P6-13 Plan - Make Default Plan Follow The Active Phase

## Objective

Stop the control plane from advertising a stale cross-phase `default_plan`, so dashboard and repo-side sync always point at the latest local slice under the currently active phase instead of an old P7 seed.

## Context

- After `P6-12`, the dashboard homepage is structurally cleaner, but the top snapshot can still claim `当前默认 Plan：P7-05 ...` because `notion_control_plane.json` keeps an old static fallback.
- This conflicts with the actual roadmap state: `P6` is still the active phase, and the local planning tree has already advanced to later P6 slices.
- The safest fix is not to manually edit `default_plan` each time. It is to derive it from the active phase in `.planning/ROADMAP.md` plus the newest local plan file under that phase, then persist the result during sync commands.

## Tasks

1. Add a repo-side helper that reads the active phase from roadmap state and resolves the newest local `PLAN.md` title under that phase.
2. Teach `gsd_notion_sync.py` entrypoints to persist that resolved plan as the effective `default_plan` before rendering dashboard / review / repo-doc snapshots.
3. Add focused tests covering both the happy path and the fallback path when no active phase plan is available.

## Success Criteria

- The dashboard `当前默认 Plan` field follows the active local P6 slice instead of a stale P7 seed.
- `notion_control_plane.json` self-heals to the new default plan during sync commands, rather than requiring manual edits.
- Existing snapshot-selection and fallback logic stay green under unit tests.
