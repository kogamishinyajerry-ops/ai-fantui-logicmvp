## Objective

Pull the Notion control-tower homepage into the same automatic refresh loop as 09C, so the main cockpit page always reflects the live phase, gate, and evidence state instead of freezing on an old manual snapshot.

## Tasks

1. Extend the Notion control-plane config and validation rules so the homepage dashboard and freeze packet are treated as first-class required pages.
2. Add a homepage rendering path to `tools/gsd_notion_sync.py` that uses the same live review snapshot as 09C.
3. Refresh the Notion homepage through that code path and verify that the stale `P1` snapshot is gone.

## Success Criteria

- The control-tower homepage reflects the current active phase, latest verified plan, latest success evidence, gate state, and current Opus posture.
- Homepage refreshes happen through the repo-side sync tool, not by hand-only editing.
- The Notion control-plane validator protects the homepage/freeze-packet page IDs as required control-plane objects.
