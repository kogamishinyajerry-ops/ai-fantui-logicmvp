# P6-09 Plan - Make Repo Entrypoints Aware Of Archived Notion Active Pages

## Objective

Stop repo-side handoff and freeze docs from pointing users at dead Notion status-style pages when the local integration can see that those targets are archived or otherwise non-writable.

## Context

- `P6-08` made the dashboard and repo docs readable again, but the local integration still sees the separate `01 当前状态` / `09C` / `10 Freeze Demo Packet` pages as archived in practice.
- Leaving those links in repo-side handoff docs makes the control plane feel less trustworthy, even though the dashboard and GitHub evidence are still current.
- The right degraded behavior is to acknowledge the archived-page constraint and steer users toward the dashboard plus GitHub until those separate pages become healthy again.

## Tasks

1. Detect archived/unwritable active-page targets during repo-doc sync.
2. Remove or replace dead Notion subpage links in repo-side coordination/freeze docs with an explicit dashboard-first degraded note.
3. Re-run repo-doc sync and validation so the handoff surface matches the real health of the Notion control plane.

## Success Criteria

- Repo-side coordination and freeze docs no longer advertise dead status/09C/freeze links when those pages are archived.
- The dashboard remains the primary live Notion entry under degraded mode, and GitHub remains the secondary evidence plane.
- Validation and repo-doc sync both pass after the degradation-aware rendering change.
