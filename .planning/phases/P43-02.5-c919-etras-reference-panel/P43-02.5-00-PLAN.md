```
Col 3 (x=360-500)  : EICU_CMD2 → EICU_CMD3 (S-R latch) → TR_COMMAND3 → FADEC_DEPLOY_CMD1 (truth-tracked logic nodes)
Col 4 (x=540-680)  : Timing guards (COMM2 / lock_unlock_confirm / position_deployed_confirm / stowed_locked_confirm · annotation hex w/ countdown) + 3 safety diamond (over-temp / vdt / power)

Col 6 (x=880-1070) : Lock-state aggregate + TR position feedback + 4 event outputs
```

### §2b · 状态模型（frontend-only · server evaluates）

**单一 truth 源**: server `evaluate_snapshot` 返回 `truth_evaluation`。前端驱动表:

| SVG element | 驱动字段 | 备注 |
|------------|---------|------|
| truth-tracked component node color | `truth_evaluation.asserted_component_values[component_id]` (bool / str) | 主路径 · 与 chat.js:1198 既有 `extractNodeStates` 语义一致 |
| truth-tracked logic node highlight | `truth_evaluation.active_logic_node_ids` (array) | 辅助高亮 · logic gate 节点 (ln_1 ~ ln_N) |
| connection line color | 由下游节点 asserted state 推断（既有 `deriveConnectionState` 模式） | 复用 · 不新建 |
| annotation timing countdown text | snapshot 原始字段 `*_confirm_s` / `comm2_timer_s` 直接 display | 装饰用 · 不参与 truth |
| annotation LGCU selector 框 | `lgcu1_*` / `lgcu2_*` 4 字段 + `_select_mlg_wow` 输出（display only） | 装饰用 · 不复刻选择逻辑 |

**node-id mapping 表** (§3 Step C 实作时创建): `C919_SVG_NODE_MAP = {svg_element_id → spec_component_or_logic_id}` · ~22 entry · 1:1 匹配 adapter spec。annotation 节点**不在** mapping 表（保证两类分离）。

**关键变化 vs v1**: v1 写 "至少 3 个绿色节点 at default snapshot" 被删。默认 snapshot (tra_deg=0 · engine_running=true · on-ground · not inhibited) 下 adapter 的实际 `asserted_component_values` 分布由 Step A smoke 实测决定 · Exit criteria 改为 "≥1 truth-tracked node 有 non-null asserted value" + "≥1 boolean asserted_component_value 与 default snapshot 语义一致 (e.g., tr_position_percent=0 → lock_state=LOCKED 可断言)"。
