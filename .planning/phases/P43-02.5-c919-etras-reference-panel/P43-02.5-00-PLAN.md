## §2. Architecture decisions

### §2a · SVG 布局策略

6-column grid · auto-flow 行高 · viewBox 1100 × 640（比 thrust-reverser 的 900×480 大约 60% · 因节点数翻倍）

```
Col 1 (x=30-160)      : Sensors (12)
Col 2 (x=200-320)     : LGCU 余度 selector (2 box · PDF 表2)
Col 3 (x=360-500)     : Command chain (EICU_CMD2 → CMD3 → TR_COMMAND3 → FADEC_CMD1)
Col 4 (x=540-680)     : Timing guards + safety limits (6 hex + 3 diamond)
Col 5 (x=720-840)     : Actuator + position feedback (2 rect + 2 indicator)
Col 6 (x=880-1070)    : State/event outputs (4)
```

节点几何形状语义（复用 chat.css 既有 class · 不新建）:
- `sensor-node-svg` (rect) → input sensors
- `logic-gate-svg` (polygon triangle) → OR/AND gates
- `actuator-node-svg` (rect slanted) → actuators
- `deploy-indicator` (small rect) → boolean outputs
- **新增 2 个 class**（§3d chat.css L1 additive · 加 rule/class 合规）:
  - `.lgcu-selector-svg` — 双框 concentric pattern 表示余度
  - `.timing-guard-svg` — 六边形 with inline countdown text

### §2b · 状态模型（frontend-only · server evaluates）

不在前端复述 c919 logic · 调用已有 server 侧 adapter:
- `POST /api/system-snapshot` (body: `{system_id: "c919-etras", snapshot: {...}}`)
- Response `nodes[].state` + `truth_evaluation.active_logic_node_ids` → 直接驱动 SVG node class
- Response `truth_evaluation.asserted_component_values` → 驱动 `.chain-node-value` text

复用 chat.js 既有 `applySnapshotToCanvas()` 模式 · 加 c919-specific 的 node-id-to-element 映射表（~30 entry · 1:1 匹配 `nodes[].id`）

### §2c · 交互绑定

12 inputs:
| Input | Control | Range / Values | Binds to |
|-------|---------|----------------|----------|
| tra_deg | slider | -32 .. 0 step 0.1 | snapshot.tra_deg |
| radio_altitude_ft | slider | 0 .. 20 step 0.5 | snapshot.radio_altitude_ft |
| n1k | slider | 0 .. 120 step 1.0 | snapshot.n1k |
| engine_running | toggle | bool | snapshot.engine_running |
| aircraft_on_ground | toggle | bool | snapshot.aircraft_on_ground |
| reverser_inhibited | toggle | bool | snapshot.reverser_inhibited |
| eec_enable | toggle | bool | snapshot.eec_enable |
| mlg_wow | derived | computed from LGCU1/LGCU2 | snapshot.lgcu1_mlg_wow_* / lgcu2_mlg_wow_* |
| lgcu1_mlg_wow_value | toggle | bool | snapshot.lgcu1_mlg_wow_value |
| lgcu1_mlg_wow_valid | toggle | bool | snapshot.lgcu1_mlg_wow_valid |
| lgcu2_mlg_wow_value | toggle | bool | snapshot.lgcu2_mlg_wow_value |
| lgcu2_mlg_wow_valid | toggle | bool | snapshot.lgcu2_mlg_wow_valid |

Event model: `oninput`/`onchange` → debounce 150ms → POST snapshot → update SVG via state dispatcher.

### §2d · Freeze 关系

C919 adapter + hardware YAML 是 frozen source (P34 + P38 · SHA-locked)。SVG 是它们的一个 view。可能 drift 的面：
- 如果将来 `c919_etras_hardware_v1.yaml` 改了，SVG hardcoded 节点 ID 或值范围可能不对齐
- **Mitigation**: Step E 加 regression test `tests/test_c919_etras_panel_schema_alignment.py` · 断言 SVG `data-node` 属性 ∈ YAML `sensor` + `logic_thresholds` + `timing` keys
