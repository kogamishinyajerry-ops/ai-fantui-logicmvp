# P9-02: Harden CI/CD Pipeline — Add Notion Sync Stage

## Context
Current GitHub Actions workflow runs validation suite but does not run the full Notion sync (run writeback + prepare-opus-review). The Opus review recommended: "CI/CD pipeline 包含 regression + validation + Notion sync 三阶段". Currently only stage 1 (regression) and stage 2 (validation) exist.

## Constitution Alignment
- From P9 exit criteria: CI/CD pipeline 包含 regression + validation + Notion sync 三阶段
- The third stage (Notion sync) should run after validation passes and update the Notion control plane

## Tasks

### Task 1: Audit current GitHub Actions workflow
- Read `.github/workflows/` to understand current CI stages
- Identify where Notion sync can be inserted

### Task 2: Add Notion sync stage to CI/CD
- After validation suite passes, run:
  ```
  python3 tools/gsd_notion_sync.py run --command "python3 tools/run_gsd_validation_suite.py" --plan "P9-02"
  python3 tools/gsd_notion_sync.py prepare-opus-review
  ```
- Add conditional: only run Notion sync if validation suite passed
- Add timeout bounds (already done in P6-16/P6-14) so slow Notion windows don't hang CI

### Task 3: Verify CI pipeline includes all three stages
- Trigger a test workflow run and confirm all three stages execute
- Confirm Notion control plane receives the CI-run evidence

## Exit Criteria
- CI/CD pipeline has 3 distinct stages: regression → validation → Notion sync
- Each stage has clear pass/fail criteria
- A failed Notion sync stage does not fail the overall pipeline (it's a writeback, not a correctness check)

## Validation
- GitHub Actions run shows all 3 stages
- Notion receives run record from the CI-run
