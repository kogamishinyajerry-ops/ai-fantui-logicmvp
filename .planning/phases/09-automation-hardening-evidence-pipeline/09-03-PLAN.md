# P9-03: Audit and Eliminate Remaining Manual Intervention Points

## Context
P9 exit criteria: GSD automation loop 从 Plan 落地到 Notion 状态更新全链路无需手动干预. After P9-01 (auto phase lifecycle) and P9-02 (CI/CD Notion sync stage), we need to audit what's left that still requires manual steps.

## Tasks

### Task 1: Audit all manual touchpoints
- Search codebase for remaining manual intervention patterns:
  - Scripts that print "请手动..." or similar
  - Steps in README/CONTRIBUTING that require manual Notion/GitHub operations
  - Any "TODO: automate this" comments
- List each touchpoint with its blocker (why it can't be automated yet)

### Task 2: Eliminate resolvable manual touchpoints
- For each touchpoint that has a clear automation path:
  - Implement the automation
  - Update docs to remove manual instructions
- Common patterns to fix:
  - Phase close requiring separate script (P9-01 addresses)
  - Notion writeback requiring manual trigger (P9-02 addresses)
  - Gap resolution requiring manual Notion edit

### Task 3: Document irreducible touchpoints
- For touchpoints that genuinely require human judgment (e.g., Opus 4.6 review):
  - Document them explicitly in the constitution/control plane
  - Ensure the GSD loop handles the handoff gracefully (degraded mode, clear message)

## Exit Criteria
- All resolvable manual touchpoints are automated
- Remaining human-only steps are explicitly documented and have clear degraded-mode handling
- GSD loop can complete a full cycle (plan → validate → Notion update → CI) without manual intervention

## Validation
- No "请手动" / "please manually" strings in automation scripts
- Full CI/CD run completes end-to-end with Notion writeback
