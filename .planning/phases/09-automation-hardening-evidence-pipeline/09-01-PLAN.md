# P9-01: Automate Roadmap DB Phase Lifecycle (Register + Close)

## Context
Opus review noted that Roadmap DB (01A) had no P7/P8 entries — Phase history was tracked only in Plan DB and QA DB. This creates a governance gap: the canonical Phase history in Roadmap DB is incomplete. P6 was also still "Active" in Roadmap DB even though it was approved earlier.

## Constitution Alignment
- Constitution: GSD automation loop 从 Plan 落地到 Notion 状态更新全链路无需手动干预
- P8-06 introduced `tools/close_p8_phase.py` as a one-off script to close P8. This is the right pattern but should be automated into the GSD loop.

## Tasks

### Task 1: Extract Phase close triggers from GSD loop
- Find where in `gsd_notion_sync.py` or `run` command a phase is naturally closed
- The natural trigger is: when a Phase's exit criteria are validated and gate is Approved
- Add a `close_phase` function that:
  - Takes `phase_name`, `phase_id` as arguments
  - Queries Roadmap DB for the phase entry
  - Updates Status from "Active" → "Done" (or creates the entry if missing)
  - Is called automatically after `prepare-opus-review` confirms Approved gate

### Task 2: Automate Phase creation on phase start
- When a new P{n} plan lands on main, the GSD loop should auto-register the phase in Roadmap DB as "Active"
- Add a function `register_new_phase(phase_id, phase_name, goal)` called from the commit hook or from `sync_repo_docs`
- The function upserts a Roadmap DB entry with Status="Active"

### Task 3: Verify Roadbounds DB now tracks P6, P7, P8 correctly
- After running, verify Roadmap DB shows:
  - P6: Done
  - P7: Done
  - P8: Done
- P9 itself should be registered as Active

## Exit Criteria
- Roadmap DB is source of truth for Phase lifecycle
- P6, P7, P8 all show Status=Done
- P9 is registered as Active
- No manual Roadmap DB edits needed for phase transitions

## Validation
- Query Roadmap DB and confirm phase statuses
- Run `python3 tools/validate_notion_control_plane.py --format json` still passes
