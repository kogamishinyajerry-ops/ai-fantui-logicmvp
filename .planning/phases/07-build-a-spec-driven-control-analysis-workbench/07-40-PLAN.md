# P7-40 Plan - Verify Archive README Self-Check Command

## Objective

Turn the archive README self-check command into executable evidence by testing it from inside a generated archive directory.

## Context

- `P7-38` documented `python3 -m well_harness.cli archive-manifest .` in generated archive README files.
- `P7-39` added the same command to machine-readable manifest metadata.
- The command should be proven as an actual working archive-directory workflow, not only asserted as a string in README/manifest output.

## Tasks

1. Generate a workbench archive in a temporary directory.
2. Run the README self-check command via subprocess from inside the generated archive directory.
3. Assert the command exits successfully and prints the expected archive validation status/self-check metadata.

## Success Criteria

- The README self-check command succeeds from the archive directory.
- The command output reports `archive_manifest: OK`.
- The output carries the same self-check command metadata that the README and manifest advertise.
