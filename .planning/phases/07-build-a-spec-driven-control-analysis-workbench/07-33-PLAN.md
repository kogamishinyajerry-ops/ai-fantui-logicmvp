# P7-33 Plan - Archive Browser Workspace Snapshot

## Objective

Make backend archive packages preserve the full recoverable browser workspace state, not just the derived handoff summary, so a saved archive can serve as both a handoff artifact and a browser-state checkpoint.

## Context

- `P7-32` already pushed browser handoff metadata into backend archive packages as `workspace_handoff.json` plus a README section.
- That still leaves the full packet/result/history/editor workspace state outside the backend archive unless the engineer separately exports a browser snapshot.
- The browser already has a stable `collectWorkbenchPacketWorkspaceState()` payload, so the archive path can persist that exact snapshot as another backend-owned artifact without inventing a parallel format.

## Tasks

1. Accept optional `workspace_snapshot` metadata in the workbench bundle API and pass it through backend archive generation.
2. Persist that payload as `workspace_snapshot.json` inside archive packages, alongside the existing handoff summary artifact.
3. Surface the new snapshot artifact in the browser archive file list and cover it with backend/demo tests.

## Success Criteria

- Archive packages can include a `workspace_snapshot.json` file when the browser supplies the current workspace snapshot.
- The browser archive file list exposes that snapshot artifact as part of the saved archive package.
- Demo/API/archive tests cover the new snapshot artifact without breaking existing archive flows.
