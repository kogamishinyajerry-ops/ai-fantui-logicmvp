# P7-59 Plan - Add Fault Diagnosis Schema Validation To Shared Suite

## Objective

Promote the new fault-diagnosis contract from focused regression coverage into the shared validation suite, so future work cannot silently drift away from the published diagnosis schema while the rest of the workbench still passes.

## Context

- `P7-58` added `fault_diagnosis_v1.schema.json` plus schema-aware diagnosis JSON output, but that protection currently lives only in focused diagnosis tests.
- Until the contract joins the shared validation suite, a future change could still regress the diagnosis JSON boundary without tripping the repo's default baseline.
- The next highest-value move is a standalone schema-validation entry point that checks real diagnosis payloads and becomes part of the repeatable repo-wide suite.

## Tasks

1. Add a standalone fault-diagnosis schema validation script that exercises real diagnosis payloads against the published schema.
2. Add regression coverage for the new validation script and update the validation-suite command inventory.
3. Re-run the full shared validation suite and confirm diagnosis schema validation now passes as a first-class shared check.

## Success Criteria

- The repo exposes a stable fault-diagnosis schema validation entry point.
- Validation coverage proves the new script passes in JSON/text mode and integrates with the shared suite inventory.
- `run_gsd_validation_suite.py --format json` passes with fault-diagnosis schema validation included.
