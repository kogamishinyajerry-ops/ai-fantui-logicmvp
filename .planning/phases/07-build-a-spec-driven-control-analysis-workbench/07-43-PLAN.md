## Context

- `P7-41` added the repo-side archive manifest JSON Schema.
- `P7-42` made generated manifests self-describing with `$schema`.
- Human archive consumers usually open `README.md` before raw JSON, so the README should also show the manifest schema reference beside the self-check command.

## Tasks

1. Add the archive manifest schema ID to the generated README Archive Manifest section.
2. Reuse the backend schema ID constant to avoid README/manifest drift.
3. Cover the README schema reference in archive regression tests.

## Success Criteria

- Generated archive README files include the archive manifest schema URL.
- The README still includes the manifest file and self-check command.
- Existing archive manifest generation and validation behavior remains unchanged.
