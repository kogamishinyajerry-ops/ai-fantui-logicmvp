# P7-28 Plan - Export And Import Browser Workspace Snapshots

## Objective

Make the browser workbench's in-progress state portable by allowing engineers to export and import full workspace snapshots instead of being limited to one browser's local persistence.

## Context

- `P7-26` and `P7-27` already keep the packet-side and result-side workspace alive across refreshes.
- That persistence is still browser-local, so engineers cannot easily move the current packet workspace, packet history, and result history across browsers or share it as a handoff artifact.
- The workbench already has a structured browser workspace snapshot, so import/export should reuse the same restore path rather than inventing a second state format.

## Tasks

1. Add browser actions to export the current workbench workspace as a JSON snapshot and import a previously exported snapshot.
2. Reuse the same workspace-restore path for browser persistence recovery and file import so packet state, result history, and replay/latest view restore consistently.
3. Keep import honest by rejecting unsupported snapshot shapes instead of silently treating arbitrary JSON as a workbench snapshot.

## Success Criteria

- Engineers can download the current workbench workspace and later restore it from a JSON snapshot file.
- Imported snapshots restore packet context, result history, and replay/latest state using the same logic as browser persistence restore.
- Invalid or unsupported snapshot files fail clearly instead of corrupting the current workbench state.
