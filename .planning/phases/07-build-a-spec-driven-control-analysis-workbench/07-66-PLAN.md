# P7-66 Plan - Publish Controller Truth Adapter Metadata Contract

## Objective

Make the controller truth adapter boundary schema-aware by publishing a machine-readable metadata contract for adapter identity and source-of-truth provenance, without introducing a second controller truth.

## Context

- `P7-52` introduced `ControllerTruthAdapter` and wrapped the reference `DeployController` behind `ReferenceDeployControllerAdapter`.
- Opus' architecture review called out controller adapter boundaries as a key generalization gap, but the adapter metadata was still an implicit dataclass shape.
- The next safe convergence step is to formalize adapter metadata before considering any broader registry or multi-adapter orchestration.

## Tasks

1. Add schema-aware serialization for `ControllerTruthMetadata`.
2. Publish `docs/json_schema/controller_truth_adapter_metadata_v1.schema.json`.
3. Add focused regression coverage proving the reference adapter metadata validates against the published schema.

## Success Criteria

- Reference controller adapter metadata exposes stable `$schema` / `kind` / `version` metadata.
- The controller adapter metadata schema documents the adapter identity/source-of-truth boundary.
- Regression coverage proves the new metadata contract without modifying `controller.py` or changing runtime controller behavior.
