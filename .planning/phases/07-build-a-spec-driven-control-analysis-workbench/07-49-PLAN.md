# P7-49 Plan - List Recent Archives In Workbench UI

## Objective

Make archive recovery discoverable inside the workbench by listing recent archive packages from the default archive root and letting users restore them with one click.

## Context

- `P7-48` already lets the browser workbench restore an archive from a pasted manifest path or archive directory.
- The remaining usability gap is discoverability: users still need to know a local path before the restore flow becomes available.
- The demo server already knows the default archive root, so the safest next slice is to surface a recent-archive list during bootstrap and reuse the existing restore action from those entries.

## Tasks

1. Add a bootstrap-side recent archive summary list derived from valid archive manifests under the default archive root.
2. Render those recent archive entries in the workbench UI with one-click restore actions.
3. Cover bootstrap summaries, UI hooks, and the new recent-archive restore affordance in regression tests.

## Success Criteria

- `GET /api/workbench/bootstrap` returns a `recent_archives` list when valid archives exist under the default archive root.
- The workbench UI shows a “最近可恢复的 Archive” section with restore buttons.
- Existing archive export, archive restore, and workbench validation behavior remain green.
