# P7-56 Plan - Publish Reusable Playback Trace Contract

## Objective

Turn deterministic playback output into a formal machine-readable contract, so second-system traces are not just runnable but also portable, schema-aware, and safe for downstream tooling to consume.

## Context

- `P7-51` published the control-system spec schema, `P7-53` published the fault taxonomy, and `P7-54` through `P7-55` proved a second system can traverse the same workbench flow and shared validation suite.
- But the playback trace itself still behaves like an implicit dataclass dump: it is machine-readable, yet it does not advertise a stable `$schema` / `kind` / `version` boundary the way the newer spec, taxonomy, and archive-manifest payloads already do.
- The next generalization-focused move is to formalize playback output too, so later validation, diffing, archive recovery, and second-system tooling can depend on one reusable trace contract instead of an undocumented Python shape.

## Tasks

1. Add explicit playback-trace schema metadata and JSON-safe serialization to `scenario_playback`.
2. Publish `docs/json_schema/playback_trace_v1.schema.json` describing the reusable playback payload.
3. Add focused regression coverage proving generated playback JSON and CLI output carry the new contract and validate against the schema when `jsonschema` is available.

## Success Criteria

- Deterministic playback JSON exposes a stable `$schema` / `kind` / `version` contract.
- `docs/json_schema/playback_trace_v1.schema.json` documents the playback payload shape.
- Regression coverage proves the generated second-system playback payload validates against the published schema.
