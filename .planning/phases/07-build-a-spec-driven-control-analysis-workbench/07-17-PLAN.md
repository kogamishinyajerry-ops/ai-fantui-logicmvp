# P7-17 Plan - Add A Side-By-Side Replay Comparison Board To The Browser Workbench

## Objective

Make historical replay easier to validate by showing the replayed result and the newest stored result side-by-side on the browser workbench.

## Context

- `P7-16` already adds a compact replay-vs-latest difference strip for verdict, scenario, fault mode, and archive state.
- That strip answers “what changed,” but it still leaves users mentally reconstructing the broader acceptance picture from the main board and the history card.
- The next smallest visual acceptance slice is to keep the replayed result and the newest result visible together in one comparison board, without adding any backend truth layer.

## Tasks

1. Add a side-by-side replay comparison board to the browser workbench that appears only during historical replay.
2. Derive both comparison columns from the existing stored history payloads so the board stays frontend-only and truth-preserving.
3. Surface the fields that matter most for acceptance: verdict, blocker, scenario, fault mode, knowledge state, and archive state.

## Success Criteria

- When a historical result is replayed, users can compare replay vs latest without leaving the main board or relying on memory.
- The board remains hidden during latest-view, preparation, running, and first-run-empty states.
- No new backend request or UI-only rules layer is introduced; the board only summarizes already stored workbench results.
