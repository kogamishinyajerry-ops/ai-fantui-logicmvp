# P7-15 Plan - Show Replay Mode And Return-To-Latest In The Browser Workbench

## Objective

Make browser-side history replay unmistakable, so users can see at a glance whether they are viewing the newest workbench result or an older replayed one.

## Context

- `P7-14` already lets users click a history card and restore an earlier result onto the main visual board.
- That solves replay, but it introduces a new UX risk: once an older result is restored, the page does not yet clearly explain that the user is no longer looking at the latest run.
- The next smallest slice is to add a replay-mode status bar plus a one-click return-to-latest action, while staying fully inside the existing browser acceptance surface.

## Tasks

1. Add a visible status bar near the main workbench verdict board that tells the user whether they are viewing the latest run or a historical replay.
2. Add a return-to-latest control that jumps the main board back to the newest stored result without forcing a rerun.
3. Keep the replay-mode state synchronized with the existing history-card selection logic, so the page never shows ambiguous viewing state.

## Success Criteria

- Users can immediately tell whether the main board is showing the latest result or an older replayed one.
- A single click returns the page to the latest available result.
- The replay-mode indicator stays in sync with history-card selection and does not require a backend change or duplicate result model.
