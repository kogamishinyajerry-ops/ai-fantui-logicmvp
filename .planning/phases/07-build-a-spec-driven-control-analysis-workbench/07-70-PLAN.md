# P7-70 Plan - Add Second System Smoke Schema Validation To Shared Suite

## Objective

Promote the second-system smoke report contract into the shared validation suite, so the non-reference generalization proof remains schema-valid in the default regression gate.

## Context

- `P7-69` published `docs/json_schema/second_system_smoke_v1.schema.json` and made `second-system-smoke --format json` schema-aware.
- The shared validation suite already runs the smoke CLI, but it does not yet validate that payload against the published schema.
- Since this smoke report is the main machine-readable proof that a second system can traverse intake -> clarification -> playback -> diagnosis -> knowledge, it should be guarded as a first-class shared check.

## Tasks

1. Add a standalone second-system-smoke schema validation script that exercises the default CLI smoke proof.
2. Add regression coverage for text/JSON/skip validator behavior.
3. Update the validation-suite command inventory and re-run focused/shared validation before Notion sync.

## Success Criteria

- The repo exposes a stable second-system-smoke schema validation entry point.
- Validation coverage proves the default CLI smoke proof validates against the published schema.
- `run_gsd_validation_suite.py --format json` passes with second-system-smoke schema validation included.
