# P7-58 Plan - Publish Reusable Fault Diagnosis Contract

## Objective

Turn deterministic fault-diagnosis output into a formal machine-readable contract, so cross-system troubleshooting artifacts are portable, schema-aware, and safe for downstream tooling to consume.

## Context

- `P7-56` and `P7-57` already formalized playback traces and promoted that contract into the shared validation baseline.
- But the fault-diagnosis layer still behaves like an implicit dataclass dump even though it is the main bridge between playback evidence and later knowledge capture.
- The next architecture-convergence move is to formalize diagnosis output too, so second-system troubleshooting results have the same explicit payload boundary as specs, playback traces, fault taxonomy, and archive manifests.

## Tasks

1. Add explicit schema metadata and contract-aware serialization to `fault_diagnosis`.
2. Publish `docs/json_schema/fault_diagnosis_v1.schema.json` describing the reusable diagnosis payload, including embedded playback traces.
3. Add focused regression coverage proving generated diagnosis JSON and CLI output carry the new contract and validate against the published schema when `jsonschema` is available.

## Success Criteria

- Deterministic fault-diagnosis JSON exposes a stable `$schema` / `kind` / `version` contract.
- `docs/json_schema/fault_diagnosis_v1.schema.json` documents the diagnosis payload shape.
- Regression coverage proves generated diagnosis payloads validate against the published schema.
