# P7-31 Plan - Copy Browser Workspace Handoff Brief

## Objective

Make the browser workbench's new handoff summary immediately shareable by letting engineers copy a concise current-state brief instead of manually retyping the same context elsewhere.

## Context

- `P7-30` already made the workspace snapshot self-describing through a handoff board and a persistent handoff note.
- Engineers still need to carry that summary into chat, docs, or tickets, and right now that means manually rewriting what the handoff board already knows.
- The workbench already computes the packet, result, archive, workspace, and note state needed for a compact text brief, so a copy action can stay fully frontend-owned.

## Tasks

1. Add a browser action to copy the current workspace handoff brief from the live handoff board.
2. Reuse the existing handoff summary truth so the copied text always matches the currently visible packet/result/archive state.
3. Keep the copied brief inclusive of the current handoff note so engineers can move both machine-derived context and human intent together.

## Success Criteria

- The workbench exposes a one-click action to copy the current handoff brief.
- The copied brief includes current packet, result, archive, workspace, and handoff note context from the live handoff board.
- Static workbench tests cover the new copy affordance and the underlying handoff-brief code path.
