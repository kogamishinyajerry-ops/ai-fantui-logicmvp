# P7-38 Plan - Add Archive README Self-Check Command

## Objective

Make each saved workbench archive package self-validating by documenting the archive manifest CLI check directly inside its generated README.

## Context

- `P7-36` added `archive-manifest` CLI validation.
- `P7-37` made the CLI accept an archive directory and show restore targets in text output.
- Archive consumers usually open `README.md` first, so the README should point them to the self-check command instead of expecting them to discover the CLI from repo history.

## Tasks

1. Add a self-check command to the archive README manifest section.
2. Keep the command directory-relative so moved archive packages remain easy to inspect.
3. Cover the README self-check instruction in archive regression tests.

## Success Criteria

- Generated archive README files include a manifest validation command.
- The command validates the current archive directory rather than hardcoding a machine-specific path.
- Existing archive manifest and README behavior remains unchanged.
