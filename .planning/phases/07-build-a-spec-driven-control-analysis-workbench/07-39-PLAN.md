# P7-39 Plan - Add Manifest Self-Check Metadata

## Objective

Make the archive self-check command machine-readable by writing it into `archive_manifest.json` as metadata, not only into the README.

## Context

- `P7-38` added the directory-relative archive manifest validation command to generated README files.
- Future restore, sync, and audit tooling should not have to parse README prose to discover the same command.
- The manifest is already the machine-readable archive map, so it should carry optional self-check metadata and validate that metadata when present.

## Tasks

1. Add manifest `self_check` metadata with the directory-relative archive validation command.
2. Validate `self_check` shape when present without invalidating older manifests that lack the field.
3. Surface `self_check` through CLI JSON/text output and cover it in regression tests.

## Success Criteria

- Generated manifests include `self_check.command` and `self_check.working_directory`.
- Invalid self-check metadata is reported by manifest validation.
- CLI output exposes the self-check command for humans and automation.
