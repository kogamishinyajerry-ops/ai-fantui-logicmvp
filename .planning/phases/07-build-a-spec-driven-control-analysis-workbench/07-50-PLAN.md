# P7-50 Plan - Refresh Recent Archive List In Place

## Objective

Let the browser workbench refresh its recent archive list without reloading the whole page, so newly created external archive packages become visible without disrupting current workbench state.

## Context

- `P7-49` added recent archive summaries to bootstrap and a one-click restore board in the workbench UI.
- That list currently refreshes only on page load or when the current browser session itself creates/restores an archive.
- Engineers may create archive packages outside the current page session, so the workbench needs a lightweight refresh path that does not reset packet drafts, history, or current result context.

## Tasks

1. Add a dedicated recent-archives API route on the demo server.
2. Add an in-place “刷新列表” action to the workbench recent archive board.
3. Cover the new API route and refresh UI hooks in regression tests.

## Success Criteria

- The demo server exposes a recent archive refresh API that returns the same recent archive summaries as bootstrap.
- The workbench UI has a visible refresh action for the recent archive board.
- Refreshing recent archives does not require a whole-page reload and existing workbench validation stays green.
