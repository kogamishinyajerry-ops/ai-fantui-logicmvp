# P7-24 Plan - Add A Packet Revision Compare Strip To The Browser Workbench

## Objective

Make restored packet revisions easier to judge by showing their high-level input-structure differences from the latest packet directly inside the browser workbench.

## Context

- `P7-23` added packet revision history and one-click restore.
- Restoring an older packet version is useful, but engineers still need a quick answer to “what is different from the latest input?”
- A lightweight compare strip should focus on packet skeleton truth, not raw JSON diff noise.

## Tasks

1. Add a compare strip for historical packet revisions versus the latest packet revision.
2. Show a compact summary of system id, document count, logic/component shape, scenario/fault coverage, and clarification answer count.
3. Hide the compare strip when the latest packet is already selected so the UI stays quiet by default.

## Success Criteria

- When a historical packet revision is selected, the browser directly shows how it differs from the latest packet.
- The compare strip focuses on engineer-meaningful input skeleton deltas instead of raw JSON noise.
- Latest-packet view stays uncluttered.
