# P7-47 Plan - Add Workbench Archive Restore API

## Objective

Expose a stable workbench API entrypoint that can reopen a saved archive package and return its archived bundle plus recoverable browser workspace metadata.

## Context

- `P7-46` already made moved archives restore-ready in backend helpers and CLI JSON output.
- The remaining gap is integration: browser or automation clients still need a stable server entrypoint instead of manually calling multiple Python loaders.
- The local demo server already owns the workbench acceptance API surface, so it is the right place to add the first archive-restore contract.

## Tasks

1. Add a backend helper that reopens `archive_manifest.json` into one restore payload containing manifest metadata, resolved files, archived bundle JSON, and optional workspace handoff/snapshot payloads.
2. Expose that restore payload through a new demo-server workbench API route.
3. Cover moved-archive restore through focused workbench-bundle and demo-server regression tests.

## Success Criteria

- A moved archive package can be reopened from `archive_manifest.json` into one structured restore payload.
- `POST /api/workbench/archive-restore` returns archived bundle metadata plus restored workspace handoff/snapshot content for a moved archive.
- Existing archive validation, bundle export, and demo workbench behavior remain green.
