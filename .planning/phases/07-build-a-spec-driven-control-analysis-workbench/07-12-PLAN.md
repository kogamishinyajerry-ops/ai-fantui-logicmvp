# P7-12 Plan - Add One-Click Visual Acceptance Presets To The Browser Workbench

## Objective

Let users run the most common workbench acceptance paths with one click, so they do not have to understand the bundle form before validating pass, block, or archive behavior.

## Context

- `P7-11` already turned the browser workbench into a visual acceptance board and pushed raw JSON into secondary debug drawers.
- The next usability gap is still workflow friction: users must first understand how to load the right sample and whether to toggle archive before they can see the visual result.
- The smallest next slice is to precompose the common flows as one-click presets, while keeping them anchored to the same backend `/api/workbench/bundle` truth and making rapid repeated clicks safe.

## Tasks

1. Add one-click preset cards for ready-with-archive, blocked-follow-up, ready-preview, and archive-retry flows.
2. Wire those presets through the existing browser workbench state so the UI can auto-load the right packet, configure archive mode, and immediately run the bundle.
3. Make the frontend treat the latest preset click as authoritative so stale responses from earlier clicks do not repaint the visible acceptance board.

## Success Criteria

- A user can run the main pass/block/archive acceptance flows without manually preparing the form first.
- The preset buttons still reuse the existing intake / bundle / archive backend truth and do not add a UI-only rules layer.
- Rapid preset clicks leave the page showing the latest intended result rather than an older delayed response.
