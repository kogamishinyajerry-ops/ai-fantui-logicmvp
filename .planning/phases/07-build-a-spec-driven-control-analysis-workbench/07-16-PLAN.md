# P7-16 Plan - Show Latest-Vs-Replay Differences In The Browser Workbench

## Objective

Make historical replay easier to understand by showing the key differences between the replayed result and the newest stored result directly on the workbench page.

## Context

- `P7-15` already tells users whether they are viewing the newest result or a replayed older result, and lets them jump back to the latest run with one click.
- The remaining comparison burden is still visual: once a historical result is reopened, the user has to remember what the latest run looked like in order to judge how it differs.
- The next smallest usability slice is to add a compact comparison strip that contrasts replay-vs-latest verdict, scenario, fault mode, and archive status without changing backend truth.

## Tasks

1. Add a comparison strip to the browser workbench that appears when the user is replaying a historical result.
2. Drive that strip from the existing history-entry payloads so it compares the replayed result against the latest stored result with no extra backend request.
3. Keep the comparison hidden during normal latest-view, preparation, and running states so the page only surfaces it when it adds value.

## Success Criteria

- Users can reopen a historical result and immediately see how it differs from the latest result.
- The comparison strip stays synchronized with the existing replay-mode state and history selection.
- No new UI-only rules layer is introduced; the strip only summarizes already stored workbench results.
