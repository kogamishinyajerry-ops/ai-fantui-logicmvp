# P7-46 Plan - Resolve Archive Files And Load Workspace Metadata

## Objective

Make portable workbench archives directly restorable by resolving manifest file references to concrete paths and loading archived browser workspace metadata from the moved archive package itself.

## Context

- `P7-45` made archive manifests location-portable by storing archive-relative file references and resolving them from `archive_manifest.json`.
- Restore or sync automation still needs two more pieces to consume a moved archive without custom glue code: a resolved file map and helper loaders for archived browser workspace metadata.
- The CLI already validates archive manifests, so it is the natural place to expose the resolved file map for downstream tooling.

## Tasks

1. Add archive helpers that resolve the manifest `files` object into absolute paths and load archived `workspace_handoff.json` / `workspace_snapshot.json` payloads from `archive_manifest.json`.
2. Extend `archive-manifest --format json` to expose the resolved file map alongside the relative manifest `files` object.
3. Cover moved-archive restore behavior with regression tests for the helper loaders and the CLI JSON output.

## Success Criteria

- A moved archive still yields a resolved absolute `bundle.json` / `README.md` / workspace artifact map from the manifest file alone.
- `load_workbench_archive_workspace_handoff()` and `load_workbench_archive_workspace_snapshot()` succeed against a moved archive package.
- CLI JSON output includes both `files` and `resolved_files`, so restore automation can consume validated archive metadata without reopening the manifest manually.
