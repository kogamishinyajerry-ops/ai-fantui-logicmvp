# P7-36 Plan - Add Archive Manifest CLI Validation

## Objective

Expose workbench archive manifest validation through the CLI so engineers and future automation can inspect archived packages without writing custom Python.

## Context

- `P7-35` added backend helpers to validate and load `archive_manifest.json`.
- The validator is useful to future restore, sync, and audit tooling, but it is still only directly available from Python.
- A small CLI surface can make archive validation part of terminal handoff, CI checks, and later automation loops.

## Tasks

1. Add an `archive-manifest` CLI command that reads a manifest path, validates it, and returns a stable exit code.
2. Support text and JSON output so humans and automation can both consume the result.
3. Cover valid and invalid generated archive manifests through CLI regression tests.

## Success Criteria

- `well_harness archive-manifest <path> --format json` returns `valid=true` and exit code `0` for generated archives.
- Missing core archive files return `valid=false`, explicit issues, and exit code `1`.
- Existing bundle/archive CLI behavior remains unchanged.
