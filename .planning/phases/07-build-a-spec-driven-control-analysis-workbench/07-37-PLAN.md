# P7-37 Plan - Improve Archive Manifest CLI UX

## Objective

Make archive manifest validation easier to run during handoff by letting the CLI accept an archive directory and by showing restore targets in the human-readable output.

## Context

- `P7-36` added the `archive-manifest` CLI command.
- The first CLI surface required users to pass the exact `archive_manifest.json` path, even though archive handoffs usually start from the archive directory.
- The JSON output already contains restore targets, but the text output should also surface them so humans can see the browser snapshot, handoff summary, and README restore anchors without opening raw JSON.

## Tasks

1. Resolve directory inputs to `<archive_dir>/archive_manifest.json` inside the `archive-manifest` command.
2. Include restore-target lines in text output.
3. Add regression coverage for directory input and restore-target text output.

## Success Criteria

- `well_harness archive-manifest <archive_dir> --format json` validates the archive's manifest and reports the resolved manifest path.
- Text output lists restore targets for human handoff.
- Existing direct-manifest-path validation behavior remains unchanged.
