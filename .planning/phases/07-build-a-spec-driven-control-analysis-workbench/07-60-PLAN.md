# P7-60 Plan - Publish Reusable Knowledge Artifact Contract

## Objective

Turn reusable knowledge artifacts into a formal machine-readable contract, so cross-system diagnosis-plus-resolution evidence can move through archive, review, and future automation workflows without relying on implicit Python structure.

## Context

- `P7-58` and `P7-59` already formalized fault-diagnosis payloads and promoted that contract into the shared validation baseline.
- But the knowledge-capture layer still behaves like an implicit dataclass dump even though it is the system's reusable memory surface for confirmed root cause, repair action, validation-after-fix, and optimization guidance.
- The next architecture-convergence move is to formalize knowledge artifacts too, so the full diagnosis-to-resolution evidence chain has one explicit payload boundary.

## Tasks

1. Add explicit schema metadata and contract-aware serialization to `knowledge_capture`.
2. Publish `docs/json_schema/knowledge_artifact_v1.schema.json` describing the reusable knowledge-artifact payload, including embedded diagnosis payloads.
3. Add focused regression coverage proving generated knowledge JSON and CLI output carry the new contract and validate against the published schema when `jsonschema` is available.

## Success Criteria

- Knowledge-artifact JSON exposes a stable `$schema` / `kind` / `version` contract.
- `docs/json_schema/knowledge_artifact_v1.schema.json` documents the knowledge payload shape.
- Regression coverage proves generated knowledge artifacts validate against the published schema.
