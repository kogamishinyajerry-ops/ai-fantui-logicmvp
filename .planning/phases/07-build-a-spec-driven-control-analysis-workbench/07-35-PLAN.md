# P7-35 Plan - Validate Workbench Archive Manifest

## Objective

Turn `archive_manifest.json` from a passive file map into a programmatically trusted restore/audit artifact by adding backend validation and loading helpers.

## Context

- `P7-34` added `archive_manifest.json` to each workbench archive package and exposed the path through backend/API/browser surfaces.
- Later restore, sync, or audit tooling should not need to trust arbitrary JSON shape just because a file is named `archive_manifest.json`.
- The archive generator already writes a stable manifest kind, version, bundle identity, file map, and restore targets, so validation can enforce that contract before future tooling consumes it.

## Tasks

1. Add backend helpers to validate manifest kind/version, archive metadata, bundle identity, file schema, restore-target consistency, and referenced file existence.
2. Add a loader that reads `archive_manifest.json`, runs the validator, and raises a clear `ValueError` on malformed manifests.
3. Cover valid and malformed archive manifests with regression tests.

## Success Criteria

- A valid generated archive manifest validates and loads without issues.
- Missing required files and malformed manifest metadata produce explicit validation failures.
- Existing archive bundle behavior remains unchanged.
