# P7-52 Plan - Introduce Reference Controller Truth Adapter Boundary

## Objective

Wrap the current reference `DeployController` behind an explicit truth-adapter interface, so runner and demo execution paths stop hardcoding `controller.py` imports while preserving the existing reference-system behavior.

## Context

- The manual Opus 4.6 architecture review identified the single `controller.py` truth path as one of the biggest blockers to system generalization.
- `P7-51` already published a schema-aware workbench spec contract, but the runtime still imported `DeployController` directly from several execution paths.
- The next smallest architecture-convergence step is not replacing the reference controller, but creating the adapter seam that future system-specific truth sources can implement.

## Tasks

1. Add a reference controller truth-adapter module with explicit metadata and delegation back to `DeployController`.
2. Update `SimulationRunner` and live demo execution helpers to consume an injected adapter instead of importing `DeployController` directly.
3. Add regression coverage that proves the adapter preserves reference outputs and that runner execution can use an injected adapter boundary.

## Success Criteria

- A dedicated adapter module wraps the current reference `DeployController` without changing its logic.
- `SimulationRunner` can execute through an injected controller adapter, and default runtime paths still preserve current behavior.
- Regression coverage proves both output parity with `DeployController` and actual adapter injection at the runner boundary.
