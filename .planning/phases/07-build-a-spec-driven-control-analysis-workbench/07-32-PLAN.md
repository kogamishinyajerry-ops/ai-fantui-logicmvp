# P7-32 Plan - Archive Browser Workspace Handoff Context

## Objective

Carry the browser workbench's handoff truth all the way into backend archive artifacts, so a saved bundle preserves the same engineer-facing current-state context that was visible in the UI at archive time.

## Context

- `P7-30` and `P7-31` made browser workspace handoff self-describing and shareable, but that context still disappeared once the archive package crossed into the backend-owned artifact set.
- The workbench archive already persists bundle, intake, clarification, playback, diagnosis, and knowledge files plus a README summary.
- The browser now has a stable derived handoff snapshot, so the archive path can store that same snapshot and expose it in the archive file list without inventing a second handoff truth source.

## Tasks

1. Accept optional browser `workspace_handoff` metadata in the workbench bundle API and pass it into backend archive generation.
2. Persist that metadata into the archive package as a dedicated JSON artifact and append a handoff section to the archive README when the metadata is present.
3. Surface the new archive handoff artifact in API/static tests and in the browser archive file list so the handoff chain remains visible end to end.

## Success Criteria

- Archive packages can include a `workspace_handoff.json` artifact when the browser supplies handoff metadata.
- Archive README files include the browser handoff summary when that metadata exists.
- The demo API and browser archive surface expose the new handoff artifact without breaking existing archive behavior.
