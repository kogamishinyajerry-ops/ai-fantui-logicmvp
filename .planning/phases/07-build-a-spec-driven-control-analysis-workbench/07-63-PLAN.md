# P7-63 Plan - Add Workbench Bundle Schema Validation To Shared Suite

## Objective

Promote the new workbench-bundle contract from focused regression coverage into the shared validation suite, so future changes cannot silently drift the main engineer-facing handoff wrapper away from its published JSON Schema.

## Context

- `P7-62` made workbench bundles schema-aware and published `docs/json_schema/workbench_bundle_v1.schema.json`.
- That contract currently has focused test coverage, but the repo-wide shared suite does not yet run a standalone workbench-bundle schema validation entry point.
- The bundle is the highest-level handoff artifact used by CLI output, archive packages, API responses, and browser workbench flows, so it should be protected beside playback, diagnosis, and knowledge contracts.

## Tasks

1. Add a standalone workbench-bundle schema validation script that exercises ready fixture, ready reference, and blocked template payloads against the published schema.
2. Add regression coverage for the new validation script and update the validation-suite command inventory.
3. Re-run focused and shared validation checks, then sync the plan through the Notion control plane.

## Success Criteria

- The repo exposes a stable workbench-bundle schema validation entry point.
- Validation coverage proves the new script passes in JSON/text mode and integrates with the shared suite inventory.
- `run_gsd_validation_suite.py --format json` passes with workbench-bundle schema validation included.
