# P7-61 Plan - Add Knowledge Artifact Schema Validation To Shared Suite

## Objective

Promote the new knowledge-artifact contract from focused regression coverage into the shared validation suite, so future work cannot silently drift away from the published artifact schema while the rest of the workbench still passes.

## Context

- `P7-60` added `knowledge_artifact_v1.schema.json` plus schema-aware knowledge JSON output, but that protection currently lives only in focused knowledge-capture tests.
- Until the contract joins the shared validation suite, a future change could still regress the knowledge JSON boundary without tripping the repo's default baseline.
- The next highest-value move is a standalone schema-validation entry point that checks real resolved knowledge artifacts and becomes part of the repeatable repo-wide suite.

## Tasks

1. Add a standalone knowledge-artifact schema validation script that exercises real resolved knowledge payloads against the published schema.
2. Add regression coverage for the new validation script and update the validation-suite command inventory.
3. Re-run the full shared validation suite and confirm knowledge-artifact schema validation now passes as a first-class shared check.

## Success Criteria

- The repo exposes a stable knowledge-artifact schema validation entry point.
- Validation coverage proves the new script passes in JSON/text mode and integrates with the shared suite inventory.
- `run_gsd_validation_suite.py --format json` passes with knowledge-artifact schema validation included.
