# P7-26 Plan - Persist The Browser Packet Workspace Across Refresh

## Objective

Keep the browser-side packet workflow alive across refreshes by restoring the current packet editor state, packet revision history, and related workbench inputs from browser storage.

## Context

- `P7-25` protects valid packet drafts before later UI actions overwrite them.
- Engineers can still lose packet work if the page refreshes or the browser session is interrupted, especially while they are editing but have not rerun yet.
- The workbench already has a clean packet-history concept, so local persistence should restore the workspace honestly without pretending invalid JSON is a valid saved revision.

## Tasks

1. Persist the current packet editor text, packet revision history, selected packet revision, and related workbench input fields into browser storage.
2. Restore that packet workspace after bootstrap, falling back to the default reference packet only when no persisted workspace exists.
3. Keep the restore path honest by preserving invalid JSON only as editor text, not as a valid saved revision.

## Success Criteria

- Refreshing the workbench restores the last packet workspace instead of always dropping back to the default reference sample.
- Packet history, selected packet context, and related form inputs survive a refresh together.
- Invalid JSON can still be recovered after refresh, but it is never silently promoted into a valid packet revision.
