# P7-41 Plan - Add Archive Manifest JSON Schema

## Objective

Document the generated `archive_manifest.json` contract as a repo-side JSON Schema so generic tooling can validate archive manifests without importing Python helpers.

## Context

- `P7-35` through `P7-40` established backend validation, CLI validation, README self-check instructions, machine-readable self-check metadata, and an executable self-check test.
- Future restore, sync, audit, or CI tooling may want to validate manifests using standard JSON Schema tooling.
- The manifest schema should align with the generated kind/version/file map/restore-target/self-check shape.

## Tasks

1. Add `docs/json_schema/workbench_archive_manifest_v1.schema.json`.
2. Cover the schema's core constants and self-check metadata shape with regression tests.
3. Optionally validate a generated manifest against the schema when `jsonschema` is installed.

## Success Criteria

- The archive manifest JSON Schema is valid JSON.
- The schema documents the manifest kind, version, files, restore targets, bundle summary, and self-check metadata.
- Generated archive manifests validate against the schema when the optional `jsonschema` dependency is available.
