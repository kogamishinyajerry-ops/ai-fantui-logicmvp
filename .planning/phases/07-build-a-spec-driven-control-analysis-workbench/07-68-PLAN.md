# P7-68 Plan - Add Workbench Archive Manifest Schema Validation To Shared Suite

## Objective

Promote the workbench archive manifest contract into the shared validation suite, so archive/restore handoff packages keep their machine-readable file map and restore targets protected by the default regression gate.

## Context

- `P7-41` through `P7-50` made workbench archives manifest-backed, portable, restorable, and discoverable.
- `docs/json_schema/workbench_archive_manifest_v1.schema.json` already documents the manifest shape, but validation still lived in focused workbench-bundle coverage.
- Since archive manifests are the machine-readable handoff boundary for saved workbench runs, they should be protected beside the bundle and core artifact contracts.

## Tasks

1. Add a standalone workbench-archive-manifest schema validation script that generates real ready and blocked archive packages.
2. Validate those manifests through both the internal manifest validator and the published JSON Schema.
3. Add regression coverage, update the validation-suite command inventory, and re-run focused/shared validation before Notion sync.

## Success Criteria

- The repo exposes a stable workbench-archive-manifest schema validation entry point.
- Validation coverage proves ready and blocked archive manifests match the published contract.
- `run_gsd_validation_suite.py --format json` passes with workbench-archive-manifest schema validation included.
