# P7-42 Plan - Add Manifest Schema Reference

## Objective

Make generated archive manifests self-describing by writing a `$schema` reference into `archive_manifest.json`.

## Context

- `P7-41` added a repo-side JSON Schema for archive manifests.
- Generated manifests should point at that schema so schema-aware tooling can discover the contract without out-of-band repo knowledge.
- Older manifests should remain acceptable when they lack `$schema`, but new manifests should validate the value when present.

## Tasks

1. Add the archive manifest schema ID to generated manifests.
2. Validate `$schema` when present without requiring it for older manifests.
3. Surface the schema reference through CLI output and update schema/tests accordingly.

## Success Criteria

- Generated `archive_manifest.json` files include the expected `$schema` URL.
- The validator flags an incorrect `$schema` when present.
- CLI JSON/text output exposes the schema reference.
