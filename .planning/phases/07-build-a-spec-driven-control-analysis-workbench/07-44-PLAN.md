# P7-44 Plan - Expose Manifest File Map In CLI JSON

## Objective

Make `archive-manifest --format json` expose the full manifest file map so future restore/sync automation can consume CLI output directly.

## Context

- `archive_manifest.json` already contains the authoritative `files` object.
- The CLI validation output exposed status, bundle summary, file count, restore targets, schema, and self-check metadata, but not the full file map.
- Automation should not have to re-open the manifest file after already calling the CLI validator.

## Tasks

1. Add the manifest `files` object to CLI JSON validation output.
2. Keep text output compact by leaving the detailed map in JSON mode.
3. Cover the CLI file-map output in regression tests.

## Success Criteria

- `well_harness archive-manifest <path> --format json` includes `files.bundle_json` and `files.summary_markdown`.
- The CLI still returns `valid=true` and exit code `0` for generated archives.
- Existing text output and validation behavior remain unchanged.
