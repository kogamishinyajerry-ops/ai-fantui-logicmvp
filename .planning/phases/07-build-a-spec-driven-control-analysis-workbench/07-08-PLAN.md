# P7-08 Plan - Archive Workbench Bundles For Handoff And Audit

## Objective

Turn the new engineer-facing workbench bundle into a durable archive artifact, so engineers can hand off, review, and later sync a complete troubleshooting package without re-running multiple CLI steps or copy-pasting stdout.

## Context

- `P7-07` already composes intake, clarification, playback, diagnosis, and knowledge capture into a single in-memory / CLI bundle.
- The remaining workflow gap is persistence. Today the bundle can be printed, but it is not yet a structured archive package that can be stored, attached to a ticket, or revisited later.
- The safest next slice is archive/export only: write the existing bundle plus its component sections to stable JSON / Markdown files without changing controller truth, playback semantics, or the knowledge schema.

## Tasks

1. Add a reusable archive helper that writes a timestamped bundle package with machine-readable JSON and a human-readable Markdown summary.
2. Extend `well_harness bundle` so engineers can emit that archive package directly from the same CLI entrypoint.
3. Add focused tests for ready and blocked bundle archives, keeping the output deterministic enough for later repo / Notion sync work.

## Success Criteria

- A bundle can be archived under a chosen root directory as a timestamped package instead of existing only as terminal output.
- The archive contains both a top-level bundle artifact and component-level JSON files that are ready for handoff, review, or future sync automation.
- Blocked clarification bundles and ready full bundles both archive cleanly without adding a second rule engine or changing existing demo / controller contracts.
