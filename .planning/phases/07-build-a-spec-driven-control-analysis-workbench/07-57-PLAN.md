# P7-57 Plan - Add Playback Trace Schema Validation To Shared Suite

## Objective

Promote the new playback-trace contract from focused regression coverage into the shared validation suite, so future work cannot silently drift away from the published playback schema while the rest of the workbench still passes.

## Context

- `P7-56` added `playback_trace_v1.schema.json` plus schema-aware playback JSON output, but that protection currently lives only in focused playback tests.
- Until the contract joins the shared validation suite, a future change could still regress the playback JSON boundary without tripping the repo's default baseline.
- The next highest-value move is a standalone schema-validation entry point that checks real playback payloads and becomes part of the repeatable repo-wide suite.

## Tasks

1. Add a standalone playback-trace schema validation script that exercises real playback payloads against the published schema.
2. Add regression coverage for the new validation script and update the validation-suite command inventory.
3. Re-run the full shared validation suite and confirm playback-trace validation now passes as a first-class shared check.

## Success Criteria

- The repo exposes a stable playback-trace schema validation entry point.
- Validation coverage proves the new script passes in JSON/text mode and integrates with the shared suite inventory.
- `run_gsd_validation_suite.py --format json` passes with playback-trace schema validation included.
