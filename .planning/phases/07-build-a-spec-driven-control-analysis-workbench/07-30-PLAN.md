# P7-30 Plan - Add Browser Workspace Handoff Notes

## Objective

Make browser-side workbench snapshots easier to hand off by attaching a concise current-state summary and an engineer-written handoff note to the portable workspace state.

## Context

- `P7-28` already made the browser workspace portable through export/import, and `P7-29` hardened the Notion-side anti-drift contract around that workflow.
- Those exported snapshots still read like raw state dumps, so the next engineer has to infer whether the packet is ready, blocked, archived, or mid-investigation.
- The workbench already has all the truth needed for a compact handoff view: current packet baseline, result history, replay/latest mode, archive state, and the user’s current editor/input context.

## Tasks

1. Add a browser-visible handoff summary board that compresses the current packet, result, archive, and workspace context into a scannable engineer-facing snapshot.
2. Add a handoff note field that persists with the browser workspace and travels with exported/imported workspace snapshots.
3. Include derived handoff metadata in exported workspace snapshots so the portable file is self-describing even before it is reloaded into the UI.

## Success Criteria

- The workbench page shows a dedicated handoff summary section with current packet, result, archive, and workspace context.
- A handoff note survives browser persistence and snapshot export/import alongside the rest of the workspace.
- Exported workspace snapshots include a derived handoff summary payload, not just the raw packet/result state.
