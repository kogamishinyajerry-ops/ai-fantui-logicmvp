## 二、Citation Drift 发现（需 Kogami 决策）

### D1 · `requirements.txt` 被 3 处 cite，文件实际不存在

本仓库用 `pyproject.toml`（无 `[project.dependencies]` section，stdlib-only）作为依赖清单，
**没有 `requirements.txt` 文件**。但 3 份在产文档里 cite 了它：

| 文件 | 行 | 当前措辞 | 推荐改法（保留语义）|
| ---- | -- | -------- | ------------------- |
| `docs/demo/faq.md` | 126 | `` `requirements.txt`（无 anthropic 依赖）`` | `` `pyproject.toml`（无 anthropic 依赖） `` |
| `docs/onboarding/README.md` | 53 | `Other runtime dependencies are listed in requirements.txt or pyproject.toml` | 去掉 "requirements.txt or"，只保留 `pyproject.toml` |
| `docs/co-development/security-review-template.md` | 30 | `新增 dependency 是否影响供应链审计？...在 requirements.txt 固定版本` | 改为 `在 pyproject.toml 固定版本` |

**为什么 P26 没拦下：** P26 的 CITATION_RE 要求路径以 `src/ tests/ scripts/ runs/ docs/ config/ data/ .planning/` 之一开头，
而 `requirements.txt` 在 repo 根目录没前缀，regex 不匹配。P28 通过硬编码 pyproject.toml
assertion 间接检出这个 drift（substantive Q9 claim 通过 pyproject.toml 验证，而非 cited file）。

**pitch 日风险评级：** 低。Q9 的**实质 claim**（runtime 不依赖 anthropic）正确。但若甲方追问
"能看下 requirements.txt 吗" 演讲者会卡 5s 才能说"是 pyproject.toml"——轻微不流畅。

**建议优先级：** 非阻塞。若 Kogami 想改，3 处全部是一行字符串替换 `requirements.txt` → `pyproject.toml`。
若维持现状，pitch 日准备演讲者在问答时直接讲 "在 pyproject.toml"。
