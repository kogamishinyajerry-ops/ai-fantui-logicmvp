Workbench Bundle 验收台

这是 P7 workbench 的验收页面：只复用 intake / clarification / playback / diagnosis / knowledge / archive 既有链路， 不改 `controller.py` 真值，也不新增第二套隐藏规则引擎。

贴 intake packet JSON，生成 engineer-facing bundle，并可选落地 archive package。

Intake -> Clarification -> Playback -> Diagnosis -> Knowledge -> Archive
返回反推逻辑演示舱 AI Document Analyzer 当前页用于验收 P7 workbench bundle / archive，不替代 demo 舱。

acceptance flow

建议你这样验收

  1. 先点“载入参考样例”，直接跑一遍 ready bundle + archive happy path。
  2. 确认右侧出现 `full_workbench_bundle`、已选 scenario / fault mode，以及 archive package 路径。
  3. 检查 playback / diagnosis / knowledge / optimization 四块结果都能直接阅读，而不是只看 raw JSON。
  4. 再点“载入空白模板”，验证 blocked clarification bundle 仍能生成并可选归档。

验收路径

  1. 1[样例] 载入样例
  2. 2[生成] 跑 Bundle
  3. 3[总览] 看结论
  4. 4[步骤] 看卡点
  5. 5[归档] 看是否留档
  6. 6[调试] 必要时再看 JSON

默认走可视化验收，不要求先读代码或原始 JSON。

[验收] 一键预设验收卡

直接点按钮就会自动载入样例并生成结果；这是默认验收入口，不要求先碰 JSON。

预设只是在复用现有 intake / bundle / archive 真实链路，不新增第二套规则。

Preset 1

通过并留档

参考样例直接跑完整 happy path，并生成 archive。

Preset 2

阻塞演示

空白模板会主动停在 clarification gate，适合看“卡在哪”。

Preset 3

快速通过

参考样例直接跑 ready bundle，但不落 archive，适合快速扫结果。

Preset 4

留档复跑

同一秒重复点也不会崩，系统会自动生成新 archive 目录。

  • 默认先看“可视化验收面板”，不是先看 Raw JSON。
  • 连续点多个预设时，以最后一次点击结果为准。
  • 只有需要排查时，才展开开发调试抽屉。
