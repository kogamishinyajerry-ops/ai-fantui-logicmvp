这是什么

这是一个 DeepSeek V4 Pro 驱动的 UI 工作台： 工程师上传或粘贴需求，模型先提出澄清问题，再生成初版逻辑图、理解批注意图， 最后给出故障注入准备和沙盒 dry-run 建议。旧画布只保留为备选模块，不参与首页主流程。

解决什么问题

当前演示要证明的是 大模型能把需求文档转成可审查的工程候选链路， 而不是把用户带进一个手工画布编辑器。所有模型产物都保持 concept/sandbox-only， 只作为工程复核输入，不构成适航合规声明。

DeepSeek V4 Pro UI 工作台 · 演示顺序

  1. 1 进入 需求理解工作台，选择 DeepSeek V4 Pro，上传或粘贴需求文本。 （展示：模型先澄清需求，而不是直接让用户画图）
  2. 2 完成澄清后进入 逻辑链路绘制，查看模型生成的候选节点、正交连线和参数面板。 （展示：初版逻辑来自模型输出，不是旧画布主入口）
  3. 3 在图纸上选择节点并提交批注，让模型理解修改意图后更新候选图纸。 （展示：DeepSeek 驱动的批注修订闭环）
  4. 4 进入 故障注入准备 和 沙盒故障注入，检查 dry-run 配置、观测点和人工审查清单。 （展示：候选输出仍保持 sandbox-only）
