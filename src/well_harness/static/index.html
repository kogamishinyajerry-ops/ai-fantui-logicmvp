FANTUI LogicMVP · 民航推力反向器控制逻辑 · 需求到验证证据工作台

面向民用航空发动机控制工程师的 AI 辅助设计验证工具。参考对象：C919 电动反推驱动系统 (E-TRAS)。 将控制逻辑需求规范与可执行验证证据直接绑定——规范变更立即触发全链路刷新，消除传统人工翻译断层。

这是什么

传统控制逻辑验证依赖工程师手工翻译规范文档为测试用例，周期长且容易遗漏。 本工作台通过 adapter 层将规范直接绑定到可执行逻辑， 提供从"反推拉杆位移"到"油门锁释放命令"的全链路信号级可视化验证， 每个逻辑条件均有 Python 真值源支撑，可追溯至 C919 FADEC 规范条款。

解决什么问题

民航适航验证的核心挑战是 需求→设计→实现 三层之间的可追溯性。 本工作台将控制逻辑的每个 AND 门条件映射到具体 adapter 接口， 支持故障注入仿真（SW 卡死/传感器失效/逻辑门固断）， 为设计评审提供可复现的工程验证证据（内部研究用途，不构成适航合规声明）。

评审演示路径 · 建议顺序

  1. 1 进入 反推逻辑工作台，点击预设「着陆展开全链路」， 观察 L1→L2→L3→L4 信号链依次激活，绿色连线代表条件满足。 （展示：正常部署逻辑完整性）
  2. 2 在「故障注入」面板勾选 SW1 stuck_off， 观察 L1 因单点失效锁死，油门锁无法释放。 （展示：安全联锁有效性 / FMEA 可验证性）
  3. 3 切换到 C919 E-TRAS 工作台，选择预设「正常展开」， 观察 EICU CMD2→CMD3→FADEC_Deploy 的双通道激活顺序。 （展示：实机架构对应关系）
  4. 4 查看 反推电路图，确认 SVG 拓扑与 controller.py::DeployController 信号一一对应。 （展示：规范→代码→可视化三层追溯）
