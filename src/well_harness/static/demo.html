[输入] 起落架控制

调节以下输入信号，观察逻辑链路的推理结论变化。

[手册] 起落架控制手册

本面板对应 `landing_gear_adapter.py` 的放下真值。控制链分为两级：先验证手柄和液压，再验证机械 uplock 释放，最后用位置与 downlock 回读确认完成。

控制目标

在手柄选择 DOWN 且液压满足门限时，打开 selector valve；uplock 释放后，驱动 extend actuator；位置达到完成门限且 downlock 结合后，判定放下完成。

关键判据

  • `gear_handle_position = DOWN`
  • `hydraulic_pressure_psi >= 2200`
  • `uplock_released = 1`
  • `gear_position_percent >= 99`
  • `downlock_engaged = 1`

读图顺序

  1. 先看手柄是否为 DOWN。
  2. 再看液压是否越过 2200 psi。
  3. 若选择阀已发令但作动器未伸展，优先检查 uplock。
  4. 伸展后仍未完成，查看位置百分比与 downlock 回读。

常见阻塞

  • 手柄未置于 DOWN。
  • 液压低于 2200 psi，L1 不通过。
  • uplock 仍未释放，L2 被阻塞。
  • 位置已接近完成但 downlock 未结合，说明机械锁定尚未闭合。

[输入] 引气活门控制

调节以下输入信号，观察逻辑链路的推理结论变化。

[手册] 引气活门控制手册

当前真值采用开阈值 35 psi、关闭迟滞地板 32 psi、过压门限 65 psi。控制单元只有在 Ready 状态下才允许开放命令；低压或过压优先导致关闭。

控制目标

保持可用引气但避免抖振和过压。入口压力健康时允许开阀；压力跌破迟滞地板或进入过压区间时，命令保持关闭。

关键判据

  • `pressure_healthy := inlet_pressure >= 35`
  • `pressure_too_low := inlet_pressure < 32`
  • `overpressure := inlet_pressure >= 65`
  • `valve_cmd := pressure_healthy AND control_unit_ready AND NOT overpressure`
  • 完成回读：`valve_position = OPEN` 且 `outlet_pressure >= 35`

读图顺序

  1. 先看入口压力区间是否处于 35 到 65 psi 之间。
  2. 再确认 `control_unit_ready` 为真。
  3. 若命令允许但阀位仍 CLOSED，优先看 `valve_position` 回读。
  4. 阀位已 OPEN 时，再看 `outlet_pressure` 是否建立到健康区间。

常见阻塞

  • 控制单元未 Ready，命令固定保持关闭。
  • 入口压力低于 32 psi，迟滞关闭路径优先。
  • 入口压力达到 65 psi 或以上，进入过压关闭路径。
  • 阀已指令打开但出口压力未建立，说明执行器或下游压降仍需排查。

[FBD] IEC 61131-3 FBD / 起落架放下控制

起落架放下真值由两级 AND 与两个阈值判据组成：先用 MUX 把手柄状态译码为布尔量，再用液压门限和 uplock 释放条件驱动命令链；完成判据单独由位置与 downlock 回读确认。

HANDLE_DOWN := MUX(sel=gear_handle_position, DOWN->1, UP->0)

SELECTOR_VALVE_CMD := HANDLE_DOWN AND (hydraulic_pressure_psi >= 2200)

EXTEND_ACTUATOR_CMD := SELECTOR_VALVE_CMD AND uplock_released

COMPLETION := EXTEND_ACTUATOR_CMD AND (gear_position_percent >= 99) AND downlock_engaged

颜色约定：青色为布尔/数据线，绿色为已使能命令路径，琥珀色为门限与比较结果。

[FBD] IEC 61131-3 FBD / 引气活门控制

引气真值带迟滞与过压保护。GE/GT 比较器先判断健康区间、低压区间和过压区间；OR 产生关闭优先级，MUX 再在关闭请求与开放使能之间选择最终 `valve_cmd`。

PRESSURE_HEALTHY := inlet_pressure >= 35

LOW_PRESSURE := 32 > inlet_pressure

CLOSE_REQ := LOW_PRESSURE OR (inlet_pressure >= 65)

VALVE_CMD := MUX(sel=CLOSE_REQ, D0=(PRESSURE_HEALTHY AND control_unit_ready), D1=0)

关闭路径优先级高于开放路径，因此 MUX 的 `SEL=close_req` 始终先裁决低压/过压状态。
